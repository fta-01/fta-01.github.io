现代密码学实验2：AES密钥推导、ECB逐字节解密与CBC位翻转攻击

课程：现代密码学实验
实验编号：EX2
实验日期：2026 年 6 月
包含实验：BAC协议AES密钥推导 / 含前缀ECB逐字节解密（Harder）/ CBC位翻转攻击

摘要

本实验报告涵盖三个密码学实验，聚焦AES分组密码在不同工作模式及应用场景下的安全性分析。实验一基于欧洲电子护照（ePassport）的基本访问控制（BAC）协议，通过分析护照机器可读区（MRZ）的编码规则，推导出缺失的校验字符，进而依据ICAO 9303标准衍生出AES-128加密密钥，解密给定的CBC模式密文。实验二源自Cryptopals Set 2 Challenge 14，在Challenge 12（字节替换ECB解密）基础上引入随机长度前缀，要求攻击者在不知道前缀长度的情况下，仍能逐字节恢复AES-128-ECB加密的目标明文字节。实验三源自Cryptopals Set 2 Challenge 16，利用AES-128-CBC模式的位翻转特性，在不知晓密钥的前提下，通过精确修改前一块密文来篡改下一块明文，构造出含有";admin=true;"的加密字符串，实现权限提升攻击。三个实验系统展示了分组密码弱点与实际攻击技术。

实验一：AES密钥编码于欧洲电子护照机器可读区（BAC协议）

背景与原理

ICAO MRZ校验位算法

校验位使用字符集编码：0-9 → 其本身数值，A-Z → 10-35，< → 0。使用权重序列 7, 3, 1 循环作用于字符编码，取加权和模10即为校验位：

CD=(∑i=0n−1wi⋅v(ci)) mod 10CD = \left(\sum_{i=0}^{n-1} w_i \cdot v(c_i)\right) \bmod 10 CD=(i=0∑n−1​wi​⋅v(ci​))mod10

其中 wi=[7,3,1]i mod 3w_i = [7, 3, 1]_{i \bmod 3}wi​=[7,3,1]imod3​，v(c)v(c)v(c) 为字符 ccc 的编码值。

BAC协议密钥衍生（ICAO 9303 Part 1 Vol 2）

K_seed=SHA-1(MRZ_info) 的 hexdigest 前32字符（16字节）\text{K\_seed} = \text{SHA-1}(\text{MRZ\_info})\ \text{的 hexdigest 前32字符（16字节）} K_seed=SHA-1(MRZ_info) 的 hexdigest 前32字符（16字节）

D_ENC=K_seed∥0x00000001\text{D\_ENC} = \text{K\_seed} \| \text{0x00000001} D_ENC=K_seed∥0x00000001

H_d=SHA-1(D_ENC)\text{H\_d} = \text{SHA-1}(\text{D\_ENC}) H_d=SHA-1(D_ENC)

ka=H_d[0:16], kb=H_d[16:32]\text{ka} = \text{H\_d}[0:16],\ \text{kb} = \text{H\_d}[16:32] ka=H_d[0:16], kb=H_d[16:32]

然后分别对 ka、kb 做 DES 奇偶校验位调整（每字节最低位置为奇校验位），最终 KENC=调整后的 ka∥调整后的 kbK_{ENC} = \text{调整后的 ka} \| \text{调整后的 kb}KENC​=调整后的 ka∥调整后的 kb（16字节）。

其中 MRZ_info\text{MRZ\_info}MRZ_info 由文件号+校验位、出生日期+校验位、有效期+校验位拼接而成。

AES-128-CBC解密

P[i]=AES_Dec(KENC,C[i])⊕C[i−1],C[0]=IV=0128P[i] = \text{AES\_Dec}(K_{ENC}, C[i]) \oplus C[i-1], \quad C[0] = IV = 0^{128} P[i]=AES_Dec(KENC​,C[i])⊕C[i−1],C[0]=IV=0128

实现步骤

Step 1：实现ICAO校验位算法，计算有效期字段 111116 的校验位，得到缺失字符 ? 为数字 7。

Step 2：提取MRZ各字段：

  • 文件号字段：12345678<（校验位 8）
  • 出生日期字段：111018（校验位 2）
  • 有效期字段：111116（校验位 7，即丢失的 ?）

构造 MRZ_info = 12345678<811101821111167（24字符），计算 SHA-1，取 hexdigest 前32字符为 K_seed。

Step 3：将 K_seed 拼接 00000001 后再次 SHA-1，得到 H_d。取 H_d 前16字符为 ka、后16字符为 kb，分别做 DES 奇偶校验位调整（每字节最低位置为奇校验位），拼接得到 16 字节 K_ENC。

Step 4：Base64解码密文，使用AES-128-CBC（IV=0x00×16）解密，去除01-00填充后输出明文口令。

背景与原理

ECB模式的核心弱点：相同明文块产生相同密文块（确定性）。

AES-ECB(P1∥P2∥⋯ )=AES(P1)∥AES(P2)∥⋯\text{AES-ECB}(P_1 \| P_2 \| \cdots) = \text{AES}(P_1) \| \text{AES}(P_2) \| \cdots AES-ECB(P1​∥P2​∥⋯)=AES(P1​)∥AES(P2​)∥⋯

前缀处理思路：

  1. 向Oracle提交2×16个相同字节 A，观察密文中出现的连续相同块
  2. 连续相同块之前的完整块即为前缀所占用的块数
  3. 逐步调整填充字节数直到相同块稳定出现，计算前缀在最后一块中占用的字节数

设前缀长度为 l=16q+rl = 16q + rl=16q+r，则需要填充 (16−r) mod 16(16 - r) \bmod 16(16−r)mod16 字节，使前缀+填充恰好对齐 q+1q+1q+1 块。

题目描述

加密Oracle将用户输入用AES-128-CBC加密，前后拼接固定字符串：

输入中的 ; 和 = 字符会被转义为 %3B 和 %3D，直接注入 ;admin=true; 无效。

解密Oracle检查解密结果中是否包含 ;admin=true;，返回 true/false。

目标：在不知道密钥的情况下，通过修改密文字节，使解密后包含管理员权限标记。

“Why does CBC mode have this property?”

背景与原理

CBC解密公式：

P[i]=AES_Dec(K,C[i])⊕C[i−1]P[i] = \text{AES\_Dec}(K, C[i]) \oplus C[i-1] P[i]=AES_Dec(K,C[i])⊕C[i−1]

位翻转原理：若修改 C[i−1]C[i-1]C[i−1] 的第 jjj 字节：

C[i−1][j]←C[i−1][j]⊕ΔC[i-1][j] \leftarrow C[i-1][j] \oplus \Delta C[i−1][j]←C[i−1][j]⊕Δ

则 P[i][j]P[i][j]P[i][j] 也会翻转同样的比特：

P′[i][j]=P[i][j]⊕ΔP'[i][j] = P[i][j] \oplus \Delta P′[i][j]=P[i][j]⊕Δ

代价是 P[i−1]P[i-1]P[i−1] 块完全被破坏（无法正确解密）。

为什么CBC有此特性？ 因为CBC解密时第 iii 块的输出 AES_Dec(C[i])\text{AES\_Dec}(C[i])AES_Dec(C[i]) 是固定的，与 C[i−1]C[i-1]C[i−1] 的XOR操作是线性的——对 C[i−1]C[i-1]C[i−1] 做的改动会直接反映在 P[i]P[i]P[i] 上。

总结与心得

实验一心得

通过本实验深刻理解了ICAO 9303标准对ePassport安全性的影响。BAC协议的设计目的是防止非接触式读取，但其密钥完全由MRZ派生，而MRZ中的信息（生日、有效期、文件号）往往并非完全随机，攻击者若能获得MRZ字段（例如肉眼观察或照片扫描），即可独立算出 KENCK_{ENC}KENC​。

本实验中"?"字符通过简单的校验位枚举即可恢复，提醒我们：密钥强度不仅依赖算法，还取决于密钥材料的熵值。

遇到的问题：密钥衍生步骤中 K_seed 应使用 hexdigest 前32字符而非 bytes 前16字节；DES 奇偶校验位调整是关键步骤，缺少该步骤会导致解密结果乱码。

实验二心得

Challenge 14相比Challenge 12的核心难点在于：随机前缀破坏了块对齐假设。通过"STIMULUS-RESPONSE"思维——向Oracle发送特定刺激，观察响应变化——可以系统性地探测前缀长度。这种方法不依赖任何密码学弱点，而是利用了ECB的确定性：相同明文块 → 相同密文块。

实验让我意识到，ECB模式的无状态性既是弱点，也可被攻击者"感知"利用。

遇到的问题：探测前缀长度时，边界情况（前缀恰好对齐16字节时）需要特别处理，否则会多计入一块。

实验三心得

CBC位翻转攻击直观展示了为什么加密不等于认证（MAC）。即使攻击者不知道密钥，也可以精确篡改明文内容，只要能修改密文。

这正是现代协议（如TLS 1.3）要求同时提供加密+认证（AEAD，如AES-GCM）的根本原因。

实验中需要精确计算字节偏移，前缀 "comment1=cooking%20MCs;userdata=" 长度恰好为32字节（两块），所以用户输入从第三块开始，修改第二块密文即可翻转第三块明文。

整体感悟

三个实验从不同角度揭示了「分组密码 ≠ 安全系统」的核心原则：

实验 核心弱点 教训
BAC ePassport 密钥熵值不足 密钥材料必须高熵
ECB逐字节（Harder） ECB确定性+工作模式 不要用ECB；前缀不能提供安全
CBC位翻转 加密不提供完整性 加密必须配合MAC/AEAD

密码学实验的价值不仅在于"破解"，更在于培养对密码系统整体安全性的系统化思维：算法选择、密钥管理、完整性保护，缺一不可。
